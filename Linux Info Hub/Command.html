What is the command line?

The command line is a method of navigating an operating system through commands issue by the keyboard.
Though not strictly required in order to use linux, it is considered the fastest and most efficient way to install software and most guides for installing software on linux are written with use of the command line in mind.
Linux isnt the only the os to use a command line, back when windows was called DOS it only offered the command line as a way to navigate the system and to this day windows can be navigated entirely in command line. It is is still so relevent and used that microsoft with windows 10 created a new command line interface called powershell. Mac os also has a command line that uses bash just like many linux distros which makes machines like the macbook a popular choice for developers.
On this page we will go over basic navigation using the command line.

Navigation

Current Location

1. user@bash: pwd (pwd stands for print working directory)
2. /home/ryan (current location is the ryan folder in the home directory)
3. user@bash

Seeing contents of current directory

1. user@bash: ls (lists the content of the directory)
2. bin Documents public_html (content of the array)
3. user@bash:
4. user@bash: ls -l (gets a more detailed list of the directory)
5. total 3 (amount of files in the directory)
6. drwxr-xr-x 2 ryan users 4096 Mar 23 13:34 bin
7. drwxr-xr-x 18 ryan users 4096 Feb 17 09:12 Documents
8. drwxr-xr-x 2 ryan users 4096 May 05 17:25 public_html

Paths

1. ls /home/ryan/Documents (lets the user see a directory's content without navigating there)
2. file1.txt file2.txt file3.txt

Changing Directory

1. user@bash: cd Documents (change to the documents directory)
2. user@bash: ls (list current directory content)
file1.txt file2.txt file3.txt (files in the documents directory)
3. user@bash: cd / (slash brings the user to the root directory)
4. user@bash: pwd (print the working directory)
5. / (shows a slash representing root)
6. user@bash: ls (list current directory content)
7. bin boot dev etc home lib var (content of the rooot directory)
8. ...
9. user@bash: cd ~/Documents (change directory to Documents)
10. pwd (print the working directory)
11. /home/ryan/Documents (shows the current directory is the documents directory)
12. cd ../.. (go back two directories)
13. user@bash: pwd (priny working directory)
14. /home (current directory)
15. user@bash: cd
16. user@bash: pwd
17. /home/ryan (current directory is the user directory)

Commands

1. /etc (Stores config files for the system)
2. /var/log (Stores log files for various system programs)
3. /bin (The location of several commonly used programs (some of which we will learn about in the rest of this tutorial)
4. /usr/bin (Another location for programs on the system)
5. pwd (Print Working Directory - ie. Where are we currently)
6. ls (List the contents of a directory)
7. cd (Change Directories - ie. move to another directory)
8. ~ (tilde) - This is a shortcut for your home directory
9. . (dot) - This is a reference to your current directory
10. .. (dotdot)- This is a reference to the parent directory
